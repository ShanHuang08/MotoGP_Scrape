# 【翻譯風格要求（非常重要）】

## 基本原則

1. 必須使用賽車語境意譯
2. 禁止逐字直譯
3. 車手訪談需翻譯成觀眾容易理解的賽評中文
4. 技術描述需符合 MotoGP 常用術語
5. 保持專業、自然、具有轉播講評風格
6. 避免機械式翻譯

---

# 【賽車語境理解規則】

## 常見術語

### Slip / Slide

| 原文 | 翻譯 |
|--------|--------|
| slip | 打滑 |
| slide | 滑動 |
| rear slide | 後輪滑動 |

依照上下文選擇最自然的說法。

# 【賽車訪談特殊理解規則】

以下類型句子禁止直譯，需轉換成賽車語境：

| 原文 | 正確理解 |
|--------|--------|
| How did you find the track? | 你覺得賽道狀況如何？ |
| Today everything was very static | 今天整場比賽的局勢沒有太大變化 |
| turn very quick and very tight | 轉向迅速且走線非常緊湊 |
| machine gun is firing rocks at you | 碎石像機關槍掃射般打向車手 |
| eco mode | 保守模式 |
| supersport mode | 全力進攻模式 |
| sport mode | 均衡模式 |

# 【輸出要求】

## 格式要求

- 直接輸出完整繁體中文翻譯
- 不要解釋
- 不要逐字對照
- 不要分析內容
- 保留所有訪談內容
- 保留原文段落結構

## 可讀性要求

- 車手回答避免過度換行
- 翻譯結果需為自然流暢的中文段落
- 必須符合 MotoGP 新聞與賽評閱讀習慣

## 大量內容處理

若一次翻譯內容過長：

- 可分成多個 Markdown 檔案
- 不可省略內容
- 不可摘要內容

---

# 【最終目標】

翻譯結果必須讓 MotoGP 車迷閱讀時感覺像：

- MotoGP 官方中文新聞
- 專業賽車媒體報導
- 轉播賽評內容

而不是機械式翻譯或逐字翻譯。
